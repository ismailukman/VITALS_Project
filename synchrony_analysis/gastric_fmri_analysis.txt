================================================================================
              GASTRIC-fMRI SYNCHRONY ANALYSIS: COMPLETE REFERENCE
================================================================================

This document describes two complementary analyses investigating synchronization
between gastric (EGG) signals and brain activity measured via fMRI:

  PART A: GASTRIC-MOTION SYNCHRONY   - Measures coupling between gastric rhythm
                                       and head motion parameters (individual +
                                       population-level analysis)

  PART B: GASTRIC-VOXEL SYNCHRONY    - Measures coupling between gastric rhythm
                                       and brain BOLD signal at each voxel
                                       (voxel-wise whole-brain analysis)

================================================================================
                          GLOSSARY OF KEY TERMS
================================================================================

SIGNAL PROCESSING TERMS:
------------------------
EGG (Electrogastrography)
    Non-invasive recording of gastric electrical activity using surface electrodes
    placed on the abdomen. Captures the slow wave rhythm of stomach contractions.

BOLD (Blood Oxygen Level Dependent) Signal
    The fMRI signal reflecting changes in blood oxygenation associated with
    neural activity. Indirect measure of brain activity.

Sampling Rate
    Number of measurements per second (Hz).
    - fMRI: 0.5 Hz (1 sample every 2 seconds, TR = 2s)
    - EGG intermediate: 10 Hz (downsampled from original ~2000 Hz)
    - Gastric rhythm: ~0.05 Hz (approximately 3 cycles per minute)

TR (Repetition Time)
    Time between successive fMRI volume acquisitions (2 seconds in this study).

Bandpass Filter
    Removes frequencies outside a specified range while preserving frequencies
    within. We use 0.02-0.08 Hz to isolate the gastric rhythm band.

FIR Filter (Finite Impulse Response)
    A type of digital filter that produces output using only current and past
    input values. Used with Hamming window for stable frequency response.

SYNCHRONY MEASURES:
-------------------
PLV (Phase Locking Value)
    Measures consistency of phase relationship between two oscillating signals
    over time. Range: [0, 1]
    - 0 = completely random phase relationship
    - 1 = perfect phase locking (phases move together perfectly)

    Formula:
        PLV = |mean(exp(i * (phase_A - phase_B)))|

    Where:
        phase_A = instantaneous phase of signal A (gastric)
        phase_B = instantaneous phase of signal B (motion/brain)

awPLV (Amplitude-Weighted Phase Locking Value)
    PLV weighted by signal amplitudes, emphasizing time points when both
    signals have high amplitude. More sensitive to physiologically meaningful
    coupling. Range: [0, 1]

    Formula:
        weights = (amp_A * amp_B) / sum(amp_A * amp_B)
        awPLV = |sum(weights * exp(i * (phase_A - phase_B)))|

    Interpretation:
        awPLV > PLV suggests coupling is stronger during high-amplitude periods

Hilbert Transform
    Mathematical operation that extracts instantaneous phase and amplitude
    (envelope) from a signal. Converts real signal to analytic signal:
        analytic_signal = signal + i * hilbert(signal)
        phase = angle(analytic_signal)
        amplitude = |analytic_signal|

STATISTICAL TERMS:
------------------
Null Distribution
    Distribution of test statistic values expected under the null hypothesis
    (no true coupling). Used to assess statistical significance.

Mismatch Null (Between-Subject)
    Null distribution created by pairing one subject's brain/motion with
    another subject's gastric signal. Tests: "Is coupling subject-specific?"

Temporal Permutation Null (Within-Subject)
    Null distribution created by circularly shifting the gastric signal in
    time (rolling). Tests: "Is coupling at this specific timing?"

p-value
    Probability of observing a result as extreme as the empirical value under
    the null hypothesis. Lower p = stronger evidence against null.

FDR (False Discovery Rate)
    Proportion of false positives among all significant findings. FDR
    correction (Benjamini-Hochberg) controls this rate at specified level
    (typically q < 0.05).

Bonferroni Correction
    Conservative multiple comparison correction. Divides alpha by number of
    tests. Controls family-wise error rate (probability of ANY false positive).

Mann-Whitney U Test
    Non-parametric test comparing two independent samples. Used for population-
    level comparison of empirical vs null distributions.

TFCE (Threshold-Free Cluster Enhancement)
    Method combining voxel intensity and spatial extent without requiring
    arbitrary cluster-defining threshold. Used in FSL Randomise.

NEUROIMAGING TERMS:
-------------------
MNI Space (Montreal Neurological Institute)
    Standard brain coordinate system for reporting locations. Allows comparison
    across subjects and studies. Resolution: 2mm isotropic.

Voxel
    3D pixel in brain imaging. Each voxel has its own timeseries. Typical
    whole-brain mask: ~250,000-280,000 voxels.

AFNI (Analysis of Functional NeuroImages)
    Software suite for fMRI preprocessing. Output files used:
    - pb05.*.scale.nii.gz: Final scaled/normalized data
    - pb04.*.blur.nii.gz: Spatially smoothed data
    - errts.*.nii.gz: Residuals after nuisance regression

FSL Randomise
    Permutation-based statistical inference tool. Performs non-parametric
    hypothesis testing without assuming specific distributions.

================================================================================
                   PART A: GASTRIC-MOTION SYNCHRONY ANALYSIS
================================================================================

PURPOSE:
--------
Investigate whether head motion during fMRI scanning is phase-locked to the
gastric rhythm. This addresses whether subjects move their heads in synchrony
with their stomach contractions (e.g., breathing-related motion coupled to
gastric rhythm).

BIOLOGICAL RATIONALE:
---------------------
- Gastric contractions may cause subtle body movements transmitted to the head
- Respiratory rhythms can couple with gastric activity
- Identifying gastric-motion coupling helps: (1) understand confounds in fMRI,
  (2) reveal gut-brain-body interactions

ANALYSIS LEVELS:
----------------
1. INDIVIDUAL LEVEL: Per-subject, per-run PLV/awPLV for each motion parameter
2. POPULATION LEVEL: Compare all subjects' empirical vs null distributions

--------------------------------------------------------------------------------
STAGE A2: COMPUTE EMPIRICAL PLV AND awPLV
--------------------------------------------------------------------------------

For each subject-run and each motion parameter:

PLV CALCULATION:
  1. Apply Hilbert transform to get phase of gastric signal
  2. Apply Hilbert transform to get phase of motion signal
  3. Compute phase difference at each timepoint
  4. PLV = |mean(exp(i * phase_difference))|

awPLV CALCULATION:
  1. Extract amplitudes from Hilbert transform
  2. Compute amplitude weights (product of amplitudes, normalized)
  3. Apply weights to phase difference vector units
  4. awPLV = |weighted sum|

CODE REFERENCE:
  ```python
  def calc_plv(signal_a, signal_b):
      a_phase = np.angle(hilbert(signal_a))
      b_phase = np.angle(hilbert(signal_b))
      plv = np.abs(np.mean(np.exp(1j * (a_phase - b_phase))))
      return plv

  def calc_awplv(signal_a, signal_b):
      analytic_a = hilbert(signal_a)
      analytic_b = hilbert(signal_b)

      phase_a = np.angle(analytic_a)
      phase_b = np.angle(analytic_b)
      amplitude_a = np.abs(analytic_a)
      amplitude_b = np.abs(analytic_b)

      phase_diff_complex = np.exp(1j * (phase_a - phase_b))
      product_amplitudes = amplitude_a * amplitude_b
      weights = product_amplitudes / np.sum(product_amplitudes)

      awplv = np.abs(np.sum(weights * phase_diff_complex))
      return awplv
  ```

EXPECTED VALUES:
  - Random coupling: PLV ~ 0.1-0.2, awPLV ~ 0.15-0.25
  - Strong coupling: PLV > 0.3, awPLV > 0.4
  - awPLV is typically higher than PLV for physiological signals

--------------------------------------------------------------------------------
STAGE A3: COMPUTE NULL DISTRIBUTION (MISMATCH)
--------------------------------------------------------------------------------

NULL HYPOTHESIS: "Coupling is not subject-specific"

PROCEDURE:
  For each subject (target):
    For each other subject (source):
      - Pair target's motion with source's gastric signal
      - Compute PLV and awPLV
      - Add to null distribution

  Result: N-1 null samples per subject (where N = total subjects)

RATIONALE:
  - If coupling is real and subject-specific, empirical values should exceed
    values obtained with mismatched (wrong subject) gastric signals
  - Controls for general properties of signals while testing specificity

CODE REFERENCE:
  ```python
  null_plvs, null_awplvs = [], []
  for (other_subj, _), other_data in all_data.items():
      if other_subj == current_subj:
          continue  # Skip same subject

      gastric_mismatch = other_data['gastric']
      min_len = min(len(motion_signal), len(gastric_mismatch))

      null_plvs.append(calc_plv(gastric_mismatch[:min_len],
                                motion_signal[:min_len]))
      null_awplvs.append(calc_awplv(gastric_mismatch[:min_len],
                                    motion_signal[:min_len]))
  ```

--------------------------------------------------------------------------------
STAGE A4: INDIVIDUAL-LEVEL STATISTICAL INFERENCE
--------------------------------------------------------------------------------

For each subject-run x motion parameter:

P-VALUE CALCULATION:
  p = proportion of null values >= empirical value

  ```python
  p_value_plv = np.mean(null_plvs_arr >= plv_empirical)
  p_value_awplv = np.mean(null_awplvs_arr >= awplv_empirical)
  ```

MULTIPLE COMPARISON CORRECTION:
  - 6 motion parameters tested per subject-run
  - PLV and awPLV tested separately (independent metrics)

  FDR Correction (Benjamini-Hochberg):
    q_values = false_discovery_control(p_values, method='bh')
    significant_fdr = q_values < 0.05

  Bonferroni Correction:
    p_bonferroni = min(p_value * 6, 1.0)
    significant_bonf = p_bonferroni < 0.05

OUTPUT COLUMNS (per row = one subject-run x motion parameter):
  - subject, run, motion_param
  - plv_empirical, awplv_empirical
  - plv_null_median, plv_null_mean, plv_null_std
  - awplv_null_median, awplv_null_mean, awplv_null_std
  - p_value_plv, p_value_awplv
  - p_fdr_plv, p_fdr_awplv
  - p_bonferroni_plv, p_bonferroni_awplv
  - sig_uncorrected_plv, sig_fdr_plv, sig_bonferroni_plv

================================================================================
                   PART B: GASTRIC-VOXEL SYNCHRONY ANALYSIS
================================================================================

PURPOSE:
--------
Map brain regions where BOLD signal fluctuations are phase-locked to the
gastric rhythm. Creates whole-brain statistical maps identifying regions
potentially involved in gut-brain communication.

BIOLOGICAL RATIONALE:
---------------------
- Interoceptive brain regions (insula, ACC, somatosensory cortex) may receive
  afferent signals from the gut
- Efferent control regions may send signals TO the gut
- Identifying gastric-coupled brain regions reveals the neural substrate of
  gut-brain axis

ANALYSIS LEVELS:
----------------
1. SUBJECT LEVEL: Per-voxel PLV/awPLV maps for each subject-run
2. GROUP LEVEL: FSL Randomise permutation testing across subjects

  ```python
  phase_diff = brain_phase - gastric_phase[np.newaxis, :]  # Broadcast
  plvs_empirical = np.abs(np.mean(np.exp(1j * phase_diff), axis=1))
  # Shape: (n_voxels,)
  ```

STEP 3: Compute empirical awPLV for each voxel
  ```python
  # Resample gastric to fMRI rate for amplitude matching
  gastric_resampled = interp1d(gastric_time_orig, gastric_signal,
                                kind='cubic')(gastric_time_new)

  # Get amplitudes
  brain_amplitude = np.abs(brain_analytic)
  gastric_amplitude = np.abs(hilbert(gastric_resampled))

  # Compute weights
  amplitude_product = brain_amplitude * gastric_amplitude[np.newaxis, :]
  weights = amplitude_product / np.sum(amplitude_product, axis=1, keepdims=True)

  # Compute awPLV
  phase_diff = np.angle(brain_analytic) - np.angle(gastric_analytic)[np.newaxis, :]
  awplvs_empirical = np.abs(np.sum(weights * np.exp(1j * phase_diff), axis=1))
  ```

  - Roll gastric signal by 1-minute increments
  - Number of permutations: k = (duration - 2*buffer) / TR
  - Typical: ~200-300 permutations

  ```python
  sample_per_min = int(10 * 60)  # 600 samples at 10 Hz
  samples_per_tr = int(10 / 0.5)  # 20 samples per TR

      # Similar for awPLV...
  ```

  ```python
  from scipy.stats import false_discovery_control

  plv_p_fdr = false_discovery_control(plv_p_vals, method='bh')
  plv_p_bonferroni = np.minimum(plv_p_vals * n_voxels, 1.0)

  sig_fdr = plv_p_fdr < 0.05
  sig_bonferroni = plv_p_bonferroni < 0.05
  ```

OUTPUT MAPS (per subject-run):
  PLV Maps:
    - plvs_empirical_*.nii.gz: Raw PLV values [0, 1]
    - plv_permut_median_*.nii.gz: Null median
    - plv_delta_*.nii.gz: Empirical - null median
    - plv_p_vals_*.nii.gz: Uncorrected p-values

  awPLV Maps:
    - awplvs_empirical_*.nii.gz

EXPECTED VALUES:

  - awplvs_empirical: mean ~0.22-0.28, std ~0.10-0.15

--------------------------------------------------------------------------------
STAGE B4: GROUP-LEVEL ANALYSIS (FSL RANDOMISE)
--------------------------------------------------------------------------------

Script: voxel_based_second_level_v2.py

PURPOSE: Perform non-parametric group-level inference using FSL Randomise
to identify brain regions with consistent gastric coupling across subjects.

STEP 1: Average within subjects (across runs)
  ```python
  for subject in subjects:
      for run in runs:
          img = nib.load(f'{measure_name}_{subject}_run{run}.nii.gz')
          imgs_runs.append(img.get_fdata())

      avg_runs = np.mean(imgs_runs, axis=0)
      nib.save(avg_runs, f'{measure_name}_{subject}_mean_runs.nii.gz')
  ```

STEP 2: Average across subjects (grand mean)
  ```python
  for subject in subjects:
      img = nib.load(f'{measure_name}_{subject}_mean_runs.nii.gz')
      imgs_subjects.append(img.get_fdata())

  grand_mean = np.mean(imgs_subjects, axis=0)
  nib.save(grand_mean, f'{measure_name}_mean_subjects.nii.gz')
  ```

STEP 3: Create design matrices for paired t-test

  Design: Compare empirical vs permutation median across subjects

  ```
  Design Matrix (design.mat):
       Contrast  Subj1  Subj2  Subj3 ... SubjN
  Emp1    1        1      0      0        0
  Emp2    1        0      1      0        0
  ...
  Null1  -1        1      0      0        0
  Null2  -1        0      1      0        0
  ...

  Contrast (design.con):
  [1, 0, 0, 0, ...]  # Test: Empirical > Null

  Group (design.grp):
  [1, 2, 3, ..., N, 1, 2, 3, ..., N]  # Exchangeability blocks
  ```

STEP 4: Create 4D input files
  ```python
  # Concatenate empirical and null median images
  image_list = []
  for measure_name in ['plvs_empirical', 'plv_permut_median']:
      for subject in subjects:
          image_list.append(nib.load(f'{measure_name}_{subject}_mean_runs.nii.gz'))

  image_4d = concat_imgs(image_list)
  nib.save(image_4d, '4d_plv.nii.gz')
  ```

STEP 5: Run FSL Randomise
  ```python
  rand = fsl.Randomise(
      in_file='4d_plv.nii.gz',
      mask='mask.nii.gz',
      tcon='design.con',
      design_mat='design.mat',
      x_block_labels='design.grp',
      base_name='result_plv_',
      vox_p_values=True,      # Voxel-wise p-values
      c_thresh=2.3,           # Cluster-forming threshold
      cm_thresh=2.3,          # Cluster mass threshold
      tfce=True,              # Threshold-Free Cluster Enhancement
      num_perm=10000          # Number of permutations
  )
  rand.run()
  ```

FSL RANDOMISE PARAMETERS:
  - num_perm=10000: Number of sign-flip permutations
  - tfce=True: Combines intensity and spatial extent
  - c_thresh=2.3: Z-score threshold for cluster definition
  - vox_p_values=True: Output voxel-wise corrected p-values

STEP 6: Run separately for PLV and awPLV
  - result_plv_*: PLV group analysis results
  - result_awplv_*: awPLV group analysis results

OUTPUT FILES:
  - result_plv_tstat1.nii.gz: T-statistic map
  - result_plv_vox_p_tstat1.nii.gz: Voxel-wise FWE-corrected p-values
  - result_plv_clustere_p_tstat1.nii.gz: Cluster-extent p-values
  - result_plv_tfce_p_tstat1.nii.gz: TFCE-corrected p-values

  Same outputs for awPLV (result_awplv_*)

INTERPRETATION:
  - Significant voxels (p < 0.05 corrected) show gastric coupling
  - Higher t-values = stronger coupling
  - TFCE p-values recommended (best balance of sensitivity/specificity)
